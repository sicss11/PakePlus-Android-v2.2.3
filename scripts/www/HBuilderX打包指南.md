# HBuilderX 打包APK完整指南

## 🔧 问题：本地资源生成失败

这个错误通常是因为缺少HBuilderX需要的图标文件或manifest配置不正确。

---

## ✅ 解决方案

### 第一步：项目准备

1. 下载并解压工程文件到本地
2. 确保项目结构如下：
```
工时管理APP/
├── index.html           # 主应用
├── manifest.json       # PWA配置（已提供）
├── sw.js              # Service Worker
├── icon.png           # 512x512 图标（已提供）
├── icon_192.png       # 192x192 图标（已提供）
└── icon_96.png        # 96x96 图标（已提供）
```

---

### 第二步：创建HBuilderX项目

1. **打开HBuilderX**

2. **新建项目**
   - 文件 → 新建 → 项目
   - 选择「5+App」
   - 项目名称：工时管理
   - 模板：默认模板
   - 点击创建

3. **导入文件**
   - 把工时管理APP文件夹中的所有文件复制到新创建的项目根目录
   - 覆盖同名文件

---

### 第三步：配置manifest.json

**重要：** HBuilderX需要配置5+App的manifest.json，不是PWA的manifest.json

1. **双击打开项目中的manifest.json**

2. **基础配置**
```json
{
  "name": "工时管理",
  "appid": "__UNI__XXXXXX",
  "description": "轻量化本地工时管理工具",
  "versionName": "1.0.0",
  "versionCode": "100"
}
```

3. **应用图标配置**
   - 找到「应用图标」配置项
   - 点击「选择图标」
   - 选择项目中的 `icon.png`（512x512）
   - HBuilderX会自动生成所有尺寸

4. **启动图配置（可选）**
   - 可以选择使用默认启动图
   - 或上传自定义启动图

5. **模块配置**
   - 勾选需要的权限：
     - Storage（存储）
     - File（文件）
     - Network（网络）

---

### 第四步：云打包

1. **打开云打包**
   - 发行 → 原生App-云打包

2. **配置打包选项**
   - 应用类型：Android
   - 使用DCloud公用证书（测试用）
   - 点击打包

3. **等待打包**
   - 通常需要3-10分钟
   - 打包完成后会显示下载链接

4. **下载APK**
   - 点击下载APK文件
   - 安装到手机测试

---

## ⚠️ 常见问题

### 问题1：本地资源生成失败

**原因：** 缺少图标文件或图标格式不对

**解决：**
1. 确保项目根目录有 `icon.png` 文件（512x512，PNG格式）
2. 重新配置manifest.json中的图标
3. 清除缓存再打包

### 问题2：打包后无法使用

**原因：** 权限配置不足

**解决：**
1. 打开manifest.json → 模块配置
2. 勾选 Storage、File、Network
3. 重新打包

### 问题3：图标显示不正常

**原因：** 图标尺寸或格式不对

**解决：**
1. 使用我们提供的 icon.png（512x512 PNG）
2. 在HBuilderX中重新配置图标

---

## 🚀 快速开始

1. 重新打包项目文件（已包含icon.png）
2. 在HBuilderX中创建5+App项目
3. 导入所有文件
4. 双击manifest.json配置图标
5. 发行 → 原生App-云打包

---

## 💡 提示

- **测试用**：可以使用DCloud公用证书
- **正式发布**：需要申请自己的证书签名
- **多设备测试**：打包后APK可以在多个安卓设备安装

如果还有问题，请检查HBuilderX控制台的错误提示！
