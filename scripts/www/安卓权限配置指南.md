# 工时管理APP - 安卓权限配置指南

## 为什么需要权限？

应用需要访问手机存储来：
- 导入CSV文件
- 导出CSV文件
- 保存数据到本地

---

## 方法一：在HBuilderX中配置（推荐）

### 第一步：打开manifest.json

在HBuilderX中，双击项目根目录下的 `manifest.json` 文件（5+App的配置文件）

### 第二步：找到"模块权限配置"

在manifest.json编辑器中，找到左侧的"模块权限配置"或"权限配置"选项卡

### 第三步：勾选文件权限

在权限列表中，找到以下权限并勾选：

```
☑ Storage - 本地存储权限
☑ File - 文件操作权限
☑ Network - 网络权限（用于PWA等）
```

或者在源码视图中添加：

```json
{
  "name": "工时管理",
  "appid": "__UNI__XXXXXX",
  "description": "轻量化本地工时管理工具",
  "versionName": "1.0.0",
  "versionCode": "100",

  "permissions": {
    "Storage": {
      "description": "本地数据存储"
    },
    "File": {
      "description": "文件读写"
    },
    "Network": {
      "description": "网络访问"
    }
  },

  "app-plus": {
    "distribute": {
      "android": {
        "permissions": [
          "<uses-permission android:name=\"android.permission.READ_EXTERNAL_STORAGE\"/>",
          "<uses-permission android:name=\"android.permission.WRITE_EXTERNAL_STORAGE\"/>",
          "<uses-permission android:name=\"android.permission.MANAGE_EXTERNAL_STORAGE\"/>",
          "<uses-permission android:name=\"android.permission.READ_MEDIA_IMAGES\"/>",
          "<uses-permission android:name=\"android.permission.READ_MEDIA_VIDEO\"/>"
        ]
      }
    }
  }
}
```

### 第四步：保存并重新打包

保存manifest.json后，重新进行云打包：
- 发行 → 原生App-云打包
- 等待打包完成
- 下载新的APK

---

## 方法二：源码方式添加权限

如果方法一不生效，可以直接编辑manifest.json源码：

```json
{
  "app-plus": {
    "distribute": {
      "android": {
        "permissions": [
          "<uses-permission android:name=\"android.permission.READ_EXTERNAL_STORAGE\"/>",
          "<uses-permission android:name=\"android.permission.WRITE_EXTERNAL_STORAGE\"/>",
          "<uses-permission android:name=\"android.permission.MANAGE_EXTERNAL_STORAGE\"/>"
        ]
      }
    }
  }
}
```

---

## 权限说明

| 权限 | 说明 | 用途 |
|-----|------|------|
| READ_EXTERNAL_STORAGE | 读取外部存储 | 导入CSV文件 |
| WRITE_EXTERNAL_STORAGE | 写入外部存储 | 导出CSV文件 |
| MANAGE_EXTERNAL_STORAGE | 管理所有文件（Android 11+） | 完整文件访问 |

---

## Android 版本差异

### Android 10 及以下
- 使用传统权限模式
- 只需要 READ_EXTERNAL_STORAGE 和 WRITE_EXTERNAL_STORAGE

### Android 11 (API 30)
- 引入分区存储（Scoped Storage）
- 默认只能访问应用专属目录
- 需要用户授权才能访问其他文件

### Android 12 及以上
- 进一步限制文件访问
- 推荐使用 MANAGE_EXTERNAL_STORAGE（特殊权限）

## 解决"导入时无法打开文件选择器"的问题

如果打包后点击导入按钮没有反应，可能是以下原因：

### 原因1：权限不足
- 确认已添加上述权限
- 重新打包APK

### 原因2：安卓版本问题
- Android 11及以上需要动态请求权限
- 在代码中添加权限请求逻辑

### 原因3：WebView配置问题
在HBuilderX的manifest.json中添加WebView配置：

```json
{
  "app-plus": {
    "distribute": {
      "android": {
        "webview": {
          "minSdkVersion": "21",
          "type": "x5"
        }
      }
    }
  }
}
```

## 动态请求权限（高级）

如果静态权限配置不够，可以在代码中动态请求：

```javascript
// 检查并请求权限
function checkAndRequestPermission() {
  // 检查是否在5+App环境中
  if (typeof plus !== 'undefined') {
    // 请求存储权限
    plus.android.requestPermissions(
      'android.permission.WRITE_EXTERNAL_STORAGE',
      function(e) {
        console.log('权限请求结果:', e);
      },
      function(e) {
        console.log('权限请求失败:', e);
      }
    );
  }
}

// 在APP启动时调用
document.addEventListener('plusready', function() {
  checkAndRequestPermission();
});
```

## 常见问题

### Q: 打包后仍然无法导入文件？
A: 检查以下几点：
1. 确认权限已添加到manifest.json
2. 重新打包APK
3. 检查手机系统版本是否为Android 11+（需要MANAGE_EXTERNAL_STORAGE）
4. 查看APK安装后的权限设置，确认存储权限已授予

### Q: 导出时保存位置不对？
A: 这是系统决定的，无法指定：
- 安卓10及以下：通常保存在Downloads目录
- 安卓11及以上：保存在应用专属目录，用户需要手动移动
- 可以在代码中提示用户文件保存位置

### Q: 需要MANAGE_EXTERNAL_STORAGE权限吗？
A:
- 一般情况下不需要
- 如果用户需要导入手机任意位置的文件，需要这个特殊权限
- 这个权限比较敏感，应用商店可能拒绝审核
